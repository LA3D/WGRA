Advantage over the PowerPoint version: no Alt-Tab to Safari, no browser-tab risk, no fallback screenshots. If Wi-Fi drops during the talk, this slide still works because everything is local.

Why Quarto for this kind of talk

Format

  • Slides authored in Markdown. The source you’re viewing is .qmd, one heading per slide.
  • Cytoscape.js loaded once at page level; any slide can drop a <div> and render into it.
  • No context switch between the story and the demo. The graph is on the slide.
  • Same source can render to PDF/Beamer (--to beamer) or PowerPoint (--to pptx) if a colleague needs a static handout.
  • Speaker notes travel with the source; they don’t get lost when you re-export.

The demo isn’t a browser window you switch to. It’s part of the slide.
